# Chaos-Monkey-Network
Эмулятор сетевых задержек и потерь пакетов


Прокси-сервер для эмуляции нестабильных сетевых условий. Он встраивается между клиентом и целевым сервером, добавляя искусственные задержки и потери пакетов.

Название проекта вдохновлено инструментом **Chaos Monkey** от Netflix (который случайно отключает сервисы в продакшене), но здесь хаос вносится на уровне сети, а не инфраструктуры.

## Практическая ценность

### 1. Тестирование отказоустойчивости приложений
Проверка поведения приложения при:
- Потере пакетов 
- Задержках  
- Отключениях сервера

### 2. Нагрузочное тестирование MQTT/HTTP брокеров
Тест брокера на потерю пакетов и задержки.


## Как это работает
+------------------+     +-------------+     +------------------+
| Приложение  | --> | Chaos Proxy | --> | Целевой сервер   |
+------------------+     +-------------+     +------------------+
                              |
                              ↓
                       задержки, потери



Прокси прозрачен для приложения — вы просто меняете адрес подключения с целевого сервера на `localhost:8888`.

## Использование

### Базовый пример (задержка 2 секунды)
```
python -m chaos_proxy start --target google.com:80 --port 8888 --delay 2
```

Теперь любой запрос к localhost:8888 будет идти к google.com с задержкой 2 секунды.
```
curl --proxy http://localhost:8888 http://google.com
```

### Добавление потери пакетов (30%)
```
python -m chaos_proxy start --target google.com:80 --port 8888 --loss 0.3
```

30% пакетов не дойдут до цели.


### Комбинация задержки и потери
```
python -m chaos_proxy start --target google.com:80 --port 8888 --delay 1 --loss 0.2
```

### Случайная задержка (0.5–3 секунды)
```
python -m chaos_proxy start --target google.com:80 --port 8888 --min-delay 0.5 --max-delay 3
```


### Фильтрация по порту (применять хаос только к порту 5432)
```
python -m chaos_proxy start --target localhost:5432 --port 8888 --delay 1 --loss 0.1
```

Поддержка правил через CLI добавится позже, сейчас правила настраиваются в коде.


### Статистика в реальном времени

При запуске с параметрами задержки и потери статистика выводится каждую секунду:
```
============================================================
Статистика Chaos Proxy (время работы: 00:02:54)
============================================================
Отправлено:   45 пакетов (12.3 KB/s)
Получено:     40 пакетов
Потеряно:     5 пакетов (11.1%)
Средняя задержка: 1023.5 мс
Всего данных: 125.4 KB отправлено, 118.2 KB получено
============================================================
```


## Команды CLI

### `start` — запуск прокси

| Аргумент | Сокращение | Тип | По умолчанию | Описание |
|----------|------------|-----|--------------|----------|
| `--target` | `-t` | строка | **обязательный** | Целевой сервер (host:port) |
| `--port` | `-p` | число | 8888 | Локальный порт для прослушивания |
| `--delay` | `-d` | число | 0.0 | Фиксированная задержка в секундах |
| `--min-delay` | - | число | 0.0 | Минимальная задержка (для случайного диапазона) |
| `--max-delay` | - | число | 0.0 | Максимальная задержка (для случайного диапазона) |
| `--loss` | `-l` | число | 0.0 | Вероятность потери пакета (0.0–1.0) |
| `--verbose` | `-v` | флаг | False | Подробное логирование |

### Остановка прокси

Комбинация `Ctrl+C`. Прокси корректно завершит все активные соединения и остановится.

## Примеры реального использования
### Проверка PostgreSQL на отказоустойчивость
```
# Запуск прокси между приложением и PostgreSQL
python -m chaos_proxy start --target localhost:5432 --port 6432 --loss 0.2

# Приложение подключается к localhost:6432
psql -h localhost -p 6432 -U user -d mydb
```

## Тестирование HTTP-клиента при задержках
```
python -m chaos_proxy start --target jsonplaceholder.typicode.com:80 --port 8888 --delay 3

#Теперь все запросы к localhost:8888 идут с задержкой 3 секунды
curl http://localhost:8888/posts/1
```

### Особенности реализации
- Асинхронность — на базе asyncio для высокой производительности
- Прозрачность — не требует изменений в клиенте (просто меняется адрес)
- Статистика — реальный мониторинг пакетов, байт, задержек, потерь
- Graceful shutdown — корректное завершение всех соединений

### Технологический стек
- Python 3.10+
- asyncio — асинхронные сокеты
- typer — интерфейс командной строки
